Debug & Inspector

Built-in tools for diagnosing layout and sync issues.

UIWeaver includes a debug overlay and inspector to help diagnose layout and sync issues without adding print statements.

Enabling the debug overlay

Press F8 while a UIWeaver screen is open (configurable in keybinds). The overlay draws colored borders around every component showing its computed bounds.

What the overlay shows

ColorMeaning
BlueContainer bounds
GreenLeaf component bounds
RedComponent overflowing its parent
YellowComponent with no computed size

Inspector

Press F9 to open the inspector panel. It shows the full component tree with:

  • Component type and ID
  • Computed bounds (x, y, width, height)
  • Visibility state
  • Bound ViewModel keys and their current values

Common issues and what to look for

SymptomCheck
Component invisibleRed or yellow border — bounds are zero or outside parent
Slot misaligned with visualSlot grid bounds vs actual slot positions
Inactive tab receiving inputInspector — visibility should be false for inactive tabs
ViewModel value not updatingInspector — check bound key current value
Layout looks wrong after resizeOverlay — check if parent clipped the child
Note: The debug overlay is client-only and has no effect in production. It can be left in the code safely.
