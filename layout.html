Layout Containers

column, row, grid, tabs, and scrollPanel.

Layout containers position their children automatically. You set constraints — padding, gap, size — and the container handles the math.

column

Stacks children vertically top to bottom.

ui.column()
    .padding(8)
    .gap(6)
    .add(ui.label(...))
    .add(ui.button(...))

row

Places children horizontally left to right.

ui.row()
    .gap(8)
    .fillWidth()
    .add(ui.energyBar())
    .add(ui.progressBar().grow())

grid

Arranges children in a fixed number of columns.

ui.grid(3, 3)   // 3 columns, 3 rows

tabs

Shows one child panel at a time with a tab bar.

ui.tabs()
    .tab("Main",  mainPanel)
    .tab("Inv",   invPanel)
    .tab("Tune",  tunePanel)

Only the active tab participates in layout and input. Inactive tabs are skipped entirely.

scrollPanel

Makes its content scrollable when it overflows the panel bounds.

ui.scrollPanel()
    .size(200, 120)
    .add(ui.column().gap(4)
        // many children
    )

Size modifiers

MethodEffect
.size(w, h)Fixed width and height
.width(w)Fixed width only
.height(h)Fixed height only
.fillWidth()Expand to parent width
.fillHeight()Expand to parent height
.grow()Fill remaining space in parent axis
.padding(n)Inner padding on all sides
.gap(n)Space between children
