Component Tree

How UIWeaver screens are structured.

Every UIWeaver screen is built from a tree of components. The root is always a single container — usually a column — and all other components are children of it.

Structure

root (column)
├── label
├── row
│   ├── energyBar
│   └── progressBar
└── button

Components are added with .add(). Containers can hold any number of children:

ui.column()
    .add(ui.label(...))
    .add(ui.row()
        .add(ui.button(...))
        .add(ui.button(...))
    );

IDs

Give components an ID with .id("name"). IDs are used for ViewModel binding, debug inspection, and server action targeting:

ui.progressBar().id("mainProgress").bind("progress")

How layout works

UIWeaver runs a layout pass before rendering. Each container measures its children and assigns them positions. You describe what the structure is — the framework decides where everything goes.

This means you never write x = leftPos + 24. If padding or a sibling changes size, everything adjusts automatically.

Lifecycle

BuildComponent tree is constructed on first open
LayoutSizes and positions are calculated
RenderEach component draws itself
InputClicks and keys are dispatched top-down
