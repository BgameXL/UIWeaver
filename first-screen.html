Your First Screen

Building a simple machine UI from scratch.

Goal

This page walks through building a minimal machine screen with a title, an energy bar, a progress bar, and a Start button.

1. Create a ViewModel

The ViewModel holds synced values the UI can read. Define it on the server side:

public class ReactorViewModel extends UIViewModel {

    public final SyncedInt energy    = syncInt("energy", 0);
    public final SyncedInt maxEnergy = syncInt("maxEnergy", 10000);
    public final SyncedInt progress  = syncInt("progress", 0);
}

2. Define the UI

Build the component tree. This runs once when the screen opens:

UIWeaver.screen(player, menu)
    .root(ui.column().padding(12).gap(8).size(300, 200)

        .add(ui.label(Component.literal("Reactor"))
            .id("title"))

        .add(ui.row().gap(8).fillWidth()
            .add(ui.energyBar()
                .bind("energy", "maxEnergy")
                .size(12, 54))

            .add(ui.progressBar()
                .id("progress")
                .fillWidth()
                .height(12))
        )

        .add(ui.button(Component.literal("Start"))
            .action("start")
            .size(60, 20))
    )
    .viewModel(reactorViewModel)
    .open();

3. Handle the server action

Actions are validated and handled on the server:

@Override
public void onAction(ServerPlayer player, String action) {
    if (action.equals("start")) {
        if (canStart()) startMachine();
    }
}

4. Update synced values

Update the ViewModel each tick and UIWeaver will sync only what changed:

@Override
public void tick() {
    viewModel.energy.set(energyContainer.getEnergyStored());
    viewModel.progress.set(currentProgress);
}
Result: A screen that automatically repositions its widgets, syncs energy and progress from the server each tick, and validates the Start action server-side before doing anything.
