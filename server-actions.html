Server Actions

How user input reaches the server safely.

Interactive widgets like buttons and toggles send named actions to the server instead of directly changing state. This keeps the client from being trusted with anything it shouldn't control.

Declaring an action

ui.button(Component.literal("Start"))
    .id("btnStart")
    .action("start")

When clicked, UIWeaver sends an action packet with the name "start" to the server.

Handling actions

Override onAction in your screen handler or machine class:

@Override
public void onAction(ServerPlayer player, String action) {
    switch (action) {
        case "start"  -> { if (canStart()) startMachine(); }
        case "stop"   -> stopMachine();
        case "reset"  -> resetMachine();
    }
}

Actions with data

Some widgets send additional data with their action:

ui.slider(0, 100)
    .id("speedSlider")
    .action("setSpeed")   // sends value as int
@Override
public void onAction(ServerPlayer player, String action, int value) {
    if (action.equals("setSpeed")) setSpeed(value);
}

Validation

Always validate on the server. The client can send any action at any time:

case "start" -> {
    if (!player.hasPermissions(2)) return;
    if (!isFormed()) return;
    if (isRunning()) return;
    startMachine();
}
