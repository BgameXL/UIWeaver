ViewModel & Binding

Syncing server state to the UI.

A ViewModel is the bridge between server state and the UI. It holds typed values that are automatically synced to the client when they change.

Defining a ViewModel

public class MachineViewModel extends UIViewModel {

    public final SyncedInt   energy    = syncInt("energy", 0);
    public final SyncedInt   maxEnergy = syncInt("maxEnergy", 10000);
    public final SyncedFloat progress  = syncFloat("progress", 0f);
    public final SyncedBool  working   = syncBool("working", false);
    public final SyncedStr   status    = syncStr("status", "Idle");
}

Synced types

TypeMethod
SyncedIntsyncInt(key, defaultValue)
SyncedFloatsyncFloat(key, defaultValue)
SyncedBoolsyncBool(key, defaultValue)
SyncedStrsyncStr(key, defaultValue)

Updating values

Set values on the server. UIWeaver detects changes and sends only the dirty values:

viewModel.energy.set(container.getEnergyStored());
viewModel.progress.set(currentProgress / maxProgress);
viewModel.working.set(isRunning());

Binding to components

Use .bind(key) to connect a component to a ViewModel value:

ui.progressBar().bind("progress")

ui.energyBar().bind("energy", "maxEnergy")

ui.label(Component.literal("")).bind("status")  // updates text

ui.toggle().bind("working")
Note: Binding is one-way — server to client. User input goes back through server actions, not through ViewModel writes.
